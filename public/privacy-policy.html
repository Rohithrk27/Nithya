Nithya Privacy Policy

Effective date: March 4, 2026

1. Scope

This policy explains how Nithya collects, uses, and stores data for app and web usage.

2. Data We Collect

  • Account data: email, auth identifiers, and profile basics.
  • Progress data: habits, quests, streaks, XP logs, punishments, and recovery plans.
  • Notification settings: reminder time, push subscription, and notification consent state.
  • Technical diagnostics required for reliability and security.
  • Web analytics data (web app only in production): basic page/performance metrics.
  • Payment verification details only when you submit them.

3. How We Use Data

  • Provide login, sync, progression, and reminders.
  • Protect service integrity and prevent abuse.
  • Improve reliability and product quality.
  • Review voluntary donation verification submissions when provided by you.

4. Data Sharing and Processors

  • No sale of personal data.
  • Data is shared only with required infrastructure providers:
  • Supabase (authentication, database, storage, and API infrastructure).
  • Vercel Analytics and Vercel Speed Insights (web app analytics/performance metrics only).
  • Data may be disclosed when legally required.

5. Donations and Payments

  • UPI contributions are optional support donations.
  • Donations do not unlock digital goods, premium features, or gameplay advantages.
  • If you submit verification, we store details you submit (amount, transaction reference, optional payer/app fields, notes, timestamps, and review status).

6. Permissions

  • INTERNET: required for sync/API access.
  • POST_NOTIFICATIONS: reminders (Android 13+ runtime permission).
  • SCHEDULE_EXACT_ALARM: optional strict reminder timing.

7. Data Retention and Deletion

  • Data is retained while your account is active and for operational/legal needs.
  • To request account/data deletion, contact itsnithyaapp@gmail.com.

8. Security

  • Production traffic uses HTTPS.
  • Release artifacts are signed before distribution.
  • SHA-256 checksums are published for install verification.

9. Policy Updates

This policy may change. The latest version and effective date are published in this repository.
